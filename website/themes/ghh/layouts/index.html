Guido's Helping Hand

In the tradition of Guido d'Arezzo, who gave music its notation

A 100% automated, open-source tool that uses computer vision and artificial intelligence to transform smartphone photographs of historical music books—Gregorian chant, antiphonaries, graduals—into clean, corrected, searchable PDFs. No manual intervention required. Built for conservationists, scholars, musicians, and anyone who cherishes the cantus.

See Live Demo View on GitHub

What It Does

A fully automated 13-stage pipeline powered by computer vision and AI that transforms smartphone photographs of fragile historical books into publication-ready digital documents—no manual intervention needed.

📷

Orientation & Perspective

Automatically detects and corrects page rotation from EXIF data, camera angle, and perspective distortion. Handles landscape, portrait, and arbitrarily rotated shots.

📑

Page Detection

Finds the page boundaries within the photograph using adaptive thresholding and edge detection, separating the book page from the background surface.

✂

Content Area Extraction

Crops to the actual content area, removing margins, binding shadows, and decorative borders. Uses ink density analysis to preserve every note and word.

📐

Deskew & Dewarping

Straightens tilted text using staff line detection and projection profiles. Corrects curved pages from book spines using polynomial mesh warping.

🎨

Image Enhancement

Color cast correction, illumination normalization, shadow removal, stain compensation, show-through suppression, and adaptive contrast enhancement—tailored for parchment and aged paper.

📄

Searchable PDF Output

Assembles all processed pages into a single PDF with optimized JPEG compression. Optional OCR layer for Latin text makes the manuscript fully searchable.

Roadmap

The core geometry pipeline is complete. Here is what comes next.

Next

Forked Pipeline Architecture

The pipeline forks after perspective correction into two branches: a Book branch (full-page processing for PDF production) and a Score branch (content-area extraction for OMR). Each branch applies its own parameters for deskew, dewarp, and enhancement.

Next

Stage 8: Dewarping

Correct page curvature from book spines using staff line polynomial mesh warping. Optional AI path using DocTr/GeoTr with OpenVINO GPU acceleration for pages with few or no staff lines.

Next

Stage 9: Enhancement

Color cast correction, illumination normalization, shadow and stain removal, show-through suppression, CLAHE contrast, denoising, and sharpening—all severity-adaptive and toggleable.

Next

Stage 10: Normalization

Cross-page color and DPI consistency. Ensures all pages in a book share a uniform appearance, matching white point and resolution across the entire volume.

Next

Stage 11: OCR

Optical character recognition for Latin text using Tesseract or Kraken. Produces an hOCR layer embedded in the final PDF, making every rubric and incipit fully searchable.

Done

Stage 12: OMR — Optical Music Recognition

Automatic transcription of square notation (neumes on 4-line staves) into machine-readable GABC notation using ChantOMR—a specialist deep learning model deployed via OpenVINO. Runs in the Score branch on content-area-cropped images for best accuracy.

Future

Stage 13: Score Render

Renders GABC transcriptions into publication-quality Gregorian notation using Gregorio, producing typeset score pages appended as an annex to the final PDF.

Done

Getting Started

Process your first book in minutes.

  • Install
    pip install ghh — or clone the repository and install in development mode with pip install -e ".[dev]".
  • Photograph your book
    Take photos of each page with a smartphone or digital camera. Any orientation, any angle—the pipeline handles it.
  • Run the pipeline
    ghh run photos/ processes every image through all stages, producing a clean PDF.
  • Inspect and compare
    ghh compare photos_output/ generates an interactive viewer to inspect every stage of the processing.
# Install from GitHub
$ pip install git+https://github.com/pgarciaq/ghh.git

# Process a book
$ ghh run my-antiphonary/

# List available stages
$ ghh stages

# Generate comparison viewer
$ ghh compare my-antiphonary_output/

# Create a publishable flipbook
$ ghh flipbook my-antiphonary_output/

# Publish for the web
$ ghh publish my-antiphonary_output/ site/ --with-flipbook
